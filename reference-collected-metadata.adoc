= NetApp Data Classificationのデータソースから収集されたメタデータ
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification は、データソースとシステムをスキャンしながらメタデータを収集します。ほとんどのメタデータは利用可能ですが、一部の属性はすべてのプロトコルで利用できるわけではありません。

[cols="15,25,25,25"]
|===
|  | *メタデータ* | *CIFS* | *NFS* 


.3+| *タイムスタンプ* | _作成時間_ | 利用可能 | 利用できません（Linuxではサポートされていません） 


| 最終アクセス日時 | 利用可能 | 利用可能 


| 最終更新日時 | 利用可能 | 利用可能 


.2+| *権限* | _オープン権限_ | 「EVERYONE」グループがファイルにアクセスできる場合、そのファイルは「組織に公開」されているとみなされます。 | 「その他」がファイルにアクセスできる場合、そのファイルは「組織に公開」されているとみなされます。 


| _ユーザー/グループアクセス_ | ユーザーとグループの情報はLDAPから取得されます | 利用できません (NFS ユーザーは通常、サーバー上でローカルに管理されるため、同じユーザーが各サーバーで異なる UID を持つことができます) 
|===
[NOTE]
====
* データ分類では、データベース データ ソースから「最終アクセス時刻」を抽出しません。
* Windows OSの旧バージョン（例えば、Windows 7やWindows 8）では、システムパフォーマンスに影響を与える可能性があるため、「最終アクセス時刻」属性の収集がデフォルトで無効になっています。この属性が収集されない場合、「最終アクセス時刻」に基づくData Classification分析に影響が出ます。必要に応じて、これらの古いWindowsシステムで最終アクセス時刻の収集を有効にすることができます。


====


== 最終アクセス時刻

NetApp Data Classificationがファイル共有からデータを抽出すると、オペレーティングシステムはそれをアクセスイベントとみなし、「最終アクセス時刻」をそれに応じて更新します。スキャン後、NetApp Data Classificationは最終アクセス時刻を元のタイムスタンプに戻そうと試みます。NetApp Data ClassificationにCIFSでの書き込み属性権限、またはNFSでの書き込み権限がない場合、システムは最終アクセス時刻を元のタイムスタンプに戻すことができません。

デフォルトでは、Data Classification は、元のタイムスタンプを復元するための書き込み権限がないファイルについてはスキャンを行いません。データの最終アクセス時刻が元のタイムスタンプにリセットされなくても問題ない場合は、システム設定ページで*書き込み権限なしでスキャン*を有効にしてください。

image:screenshot_scan_missing_permissions.png["最終アクセス時刻を元のタイムスタンプに戻すための適切な権限がない場合でも、Data Classification がファイルをスキャンできるようにするために使用できるスイッチを示すスクリーンショット。"]

この機能は、オンプレミスのONTAPシステム、 Cloud Volumes ONTAP、 Azure NetApp Files、 Amazon FSx for NetApp ONTAP管理、およびサードパーティのファイル共有に適用できます。


NOTE: SnapLock で構成された ONTAP ボリュームは読み取り専用権限を持ち、最終アクセス時刻を元のタイムスタンプに戻すこともできません。



=== スキャン分析イベント

調査ページには「スキャン分析イベント」というフィルターがあり、NetApp Data Classificationで最終アクセス時刻を元に戻せなかったために分類されなかったファイル、またはNetApp Data Classificationで最終アクセス時刻を元に戻せなかったにもかかわらず分類されたファイルを表示できます。

image:screenshot_scan_analysis_event_filter.png["書き込み権限の有無に基づいて、スキャンされたファイルとスキャンされなかったファイルを確認するためのフィルターのスクリーンショット。"]

フィルターの選択肢は次のとおりです。

* 「分類されていません -- 最終アクセス時刻を元に戻すことができません」 - これは、書き込み権限が不足しているために分類されなかったファイルを示します。
* 「分類され、最終アクセス時刻が更新されました」 - これは、分類されたファイルのうち、NetApp Data Classificationによって最終アクセス時刻を元の日付にリセットできなかったファイルを示します。このフィルターは、*書き込み権限なしでスキャン*が有効になっている環境でのみ有効です。


必要に応じて、これらの結果をレポートとしてエクスポートすることで、権限の関係でスキャンされているファイルとスキャンされていないファイルを確認できます。link:task-investigate-data.html#download-your-report["データ調査レポートの詳細"^]。
